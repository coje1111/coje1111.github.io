# 2026-01-22

## 오늘 한 것
- 지금 내 학습 상태를 솔직하게 점검함
- “왜 코드를 못 쓰는지”를 능력 문제가 아니라 사고 흐름 문제로 인식
- Java 기초를 다시 외우는 게 아니라, 문제를 보고 생각부터 하는 훈련이 필요하다는 점 정리
- 완벽하게 이해하려는 태도가 오히려 진도를 막고 있었다는 걸 자각
- 하루 복습 시간을 늘리는 대신, 짧게라도 사고 훈련을 꾸준히 하는 방향으로 기준 수정
- 힌트 없이 문제를 보고 스스로 생각하는 훈련을 시작하기로 결정

## 막힌 점
- 문제를 보면 어떤 코드부터 써야 할지 머릿속에 떠오르지 않음
- “아는 문법”과 “실제로 쓰는 코드” 사이의 간극이 큼
- 복습을 하면 완벽하게 알고 싶어져서 오히려 시작이 늦어짐
- 하루 공부 시간이 부족하다는 압박 때문에 계속 조급해짐

## 정리
- 지금은 구현 실력보다 사고 흐름을 만드는 단계라는 걸 받아들임
- 코드를 바로 쓰지 못해도, 문제를 말로 풀 수 있으면 정상이라는 기준을 세움
- 완벽한 이해는 지금 목표가 아니라는 점을 명확히 함
- 당분간은 짧은 시간이라도 사고 훈련을 최우선으로 유지하기로 함
- “아무것도 못 한다”는 감정이 사실이 아니라 착각일 수 있다는 걸 인지

## 코드
- 코드 작성 없음
- 사고 훈련 문제 풀이 위주로 학습
