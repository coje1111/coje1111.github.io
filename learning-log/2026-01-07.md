# 2026-01-07

## 오늘 한 것
- 조건문(if / else if / else)을 활용한 결제 금액 계산 문제 1개 풀이
- 할인 → 배송비 → 포인트 차감 순서로 계산 흐름 정리
- 여러 조건 중 하나만 적용되도록 조건 우선순위 점검
- 계산 결과를 출력해서 로직이 맞는지 직접 확인

## 막힌 점
- 문제 구조 자체는 어제와 거의 동일해서 새로운 사고가 많이 필요하지는 않았음
- 조건을 “왜 이렇게 나눴는지”보다는 이전 패턴대로 처리하게 됨
- 문제 변형이 적어 연습 효과가 크다고 느껴지진 않았음

## 해결
- 계산 순서를 고정해서 실수 없이 처리하는 데 집중
- 조건문이 길어질수록 가독성이 중요하다는 점을 다시 인식
- 오늘은 잔디 목적 + 흐름 유지용으로 마무리하기로 판단
